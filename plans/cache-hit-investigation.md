# 缓存命中为 0% 调查计划

## 现象

pi 插件安装后，偶尔出现缓存命中率为 0% 的情况。

## 假设（按优先级排序）

1. **`pi-editor-shell` 算错缓存数据** — 缓存命中率实际并非 0%，只是显示错误
2. **某插件破坏了缓存 key** — 可能修改了 `session-id`、消息文本、系统提示词等缓存 key 相关字段，导致缓存永远无法命中。需找出具体是哪个插件
3. **缓存时间过期** — 等待太久导致缓存自然失效
4. **其他插件干扰或正常现象** — 如超时、第三方插件干扰等
5. **算法与回话回溯的冲突** - 当回溯发生（跳转到之前的对话节点并继续对话）时，实际依旧能获得缓存效果，但计算方法得到的是 0%

## 调查步骤

### 1. 确认问题真实存在

- [ ] 观察缓存命中为 0% 时的具体场景（何种操作后出现、复现频率）
- [ ] 确认此时是否真的有缓存数据可命中

### 2. 排查 pi-editor-shell

- [ ] 审查缓存数据计算/显示逻辑
- [ ] 在 0% 时手动比对真实缓存命中情况

### 3. 排查插件对缓存 key 的影响

检查各插件在 session 中的操作，重点看：
- [ ] 是否修改了 `session-id`
- [ ] 是否修改了消息文本（如注入/过滤消息）
- [ ] 是否修改了系统提示词
- [ ] 是否干预了缓存 key 的生成逻辑

涉及候选插件：
- `pi-editor-shell`
- 其他所有已装插件（逐一排查）

### 4. 排除自然过期

- [ ] 确认缓存 TTL 设置
- [ ] 对比 0% 出现的时间间隔与 TTL 的关系

### 5. 第三方插件/正常显现

- [ ] 排查其他非本仓库插件的影响
- [ ] 确认是否为正常的缓存未命中（首次调用、超时等）
